Input and Output in C

Communicating with your program using `printf` and `scanf`.

Understanding `printf()`: Formatted Output

We've seen printf() used to print a simple string. Its real power comes from format specifiers. These are placeholders that you replace with the values of your variables.

A format specifier starts with a % and tells printf() *what kind* of data to print.

int age = 25;
float pi = 3.14159;

// %d is the placeholder for an integer
printf("My age is %d.\n", age);

// %f is the placeholder for a float
printf("The value of PI is %f.\n", pi);

// You can print multiple variables at once
printf("I am %d years old and the value of PI is %f.\n", age, pi);

Output:
My age is 25. The value of PI is 3.141590. I am 25 years old and the value of PI is 3.141590.

Common Format Specifiers

You must use the correct specifier for your variable's data type, or you will get unexpected results!

  • %d or %i: For int (integer)
  • %f: For float (floating-point number)
  • %lf: For double (long float / double-precision)
  • %c: For char (single character)
  • %s: For a string (array of characters, which we'll cover later)
  • %%: To print an actual percent sign %.

You can also control formatting. For example, %.2f tells a float or double to print only two decimal places. 

printf("PI is %.2f\n", 3.14159); // Output: PI is 3.14

Understanding `scanf()`: Getting User Input

scanf() is the "scan formatted" function, and it's the partner to printf(). It reads input from the user at the console.

It uses the same format specifiers to know what kind of data to *read*.

CRITICAL SYNTAX: You must pass the memory address of the variable where you want to store the input. You do this using the ampersand &, which is the "address-of" operator. 

int user_age;

printf("Please enter your age: ");

// 1. Use %d to read an integer
// 2. Use &user_age to tell scanf WHERE to store the integer
scanf("%d", &user_age);

printf("You are %d years old.\n", user_age);

Forgetting the & in scanf() is the most common bug for C beginners and will cause your program to crash.

Example: Real-time Input/Output

Let's combine this into a program that asks for two pieces of information.

#include <stdio.h>

int main() {
    int age;
    char initial;
    double height;

    // --- Get Age ---
    printf("Enter your age: ");
    scanf("%d", &age);

    // --- Get Initial ---
    printf("Enter your first initial: ");
    // Note the space before %c. This is important!
    // It tells scanf to skip any leftover whitespace (like the Enter key)
    // from the previous input.
    scanf(" %c", &initial);

    // --- Get Height (in meters) ---
    printf("Enter your height in meters (e.g., 1.75): ");
    // Use %lf to read a double!
    scanf("%lf", &height);

    // --- Print Summary ---
    printf("\n--- Your Profile ---\n");
    printf("Initial: %c\n", initial);
    printf("Age: %d\n", age);
    printf("Height: %.2f meters\n", height);

    return 0;
}

Exercise: Type, compile, and run this program. Try entering your age, initial, and height. Does it work as expected?
