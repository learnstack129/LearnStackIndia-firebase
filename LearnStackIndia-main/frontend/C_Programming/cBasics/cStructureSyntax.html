Structure and Syntax of a C Program

The anatomy of a basic C file.

Anatomy of a C Program

At its core, every C program is a collection of functions and variables. Let's break down a minimal program:


// 1. Preprocessor Directive
#include <stdio.h>

// 2. The Main Function
int main() {
    // 3. A Statement
    printf("Hello, world!");

    // 4. Return Statement
    return 0;
} // 5. Block End
  • Preprocessor Directive (#include): Tells the compiler to include the stdio.h (Standard Input/Output) file. This file contains declarations for functions like printf().
  • Main Function (int main()): This is the entry point of every C program. Execution *always* begins here. The int means it returns an integer value.
  • Statement (printf(...)): A single instruction. This statement calls the "print formatted" function to display text on the screen.
  • Return Statement (return 0;): Exits the main() function and sends a status code (0 means success) back to the operating system.
  • Blocks ({ ... }): Curly braces define a "block" or a group of statements. Here, they define the body of the main() function.

Statements, Blocks, and Semicolons

C's syntax relies on a few simple rules:

  • Semicolons (;): The semicolon is a statement terminator. It's like the period at the end of a sentence. It tells the compiler where one instruction ends and the next begins.
    int x = 5; (This is one statement)
  • Blocks ({ ... }): Curly braces group one or more statements into a single block. Blocks are used for function bodies, loops (for, while), and conditionals (if, else). A block is a "compound statement" and does *not* end with a semicolon.
  • Statements: A statement is a complete instruction given to the computer. Most statements in C are simple (like x = 10;) or compound (a block {...}).

Comments and Formatting Standards

Code must be readable by humans, not just computers. Comments and good formatting are essential.

Comments are ignored by the compiler and are used to leave notes for yourself or other programmers.

  • Single-Line Comments: Start with // and go to the end of the line.
    // This is a single-line comment
  • Multi-Line Comments: Start with /* and end with */. They can span multiple lines. 
    
/*
  This is a
  multi-line comment.
*/

Formatting (Whitespace): C is a "free-form" language. It doesn't care about spaces, tabs, or newlines (collectively, "whitespace") in most cases.

This: int x=5; is the same as This: int x = 5 ;

Use whitespace to make your code neat and readable. Indent code inside blocks (like main()) to show structure. This is a critical formatting standard.
